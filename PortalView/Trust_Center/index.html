This page provides transparency about how PortalView handles data, security, and compliance requirements when used within Atlassian Cloud.

Privacy Policy

PortalView is an Atlassian Forge application designed to display Jira Service Management fields, SLA information, and Assets data directly in the customer portal.

The app does not collect, store, or process personal data outside of Atlassian’s infrastructure.

Data access

  • Jira issues and fields explicitly configured by administrators
  • Jira Service Management request and request type information
  • SLA status and timing data
  • Assets objects and their attribute values (when enabled)

Data storage

The app stores configuration data only, such as Project ID, Request Type ID, selected fields, and SLA configuration. All configuration data is stored securely using Atlassian Forge Storage, scoped to the customer’s Jira instance.

Data sharing

PortalView does not share, sell, or transmit data to third parties.

Security Statement

PortalView is built on the Atlassian Forge platform and inherits Atlassian’s security controls, isolation model, and compliance standards.

  • Runs entirely within Atlassian Cloud infrastructure
  • No external servers or APIs are used
  • No outbound network calls
  • No custom authentication mechanisms
  • Permissions follow the principle of least privilege

All access to Jira and Jira Service Management data is governed by Atlassian’s permission model and the end user’s access rights.

Data Residency

PortalView follows Atlassian Forge data residency rules. All data processed and stored by the app remains within Atlassian-controlled environments and respects the data residency location configured for the customer’s Atlassian Cloud instance.

Compliance

PortalView is designed to support compliance with GDPR and LGPD through data minimization, purpose limitation, access control based on Jira permissions, and automatic deletion of stored configuration data upon app uninstallation.

PortalView does not independently certify compliance standards such as ISO 27001 or SOC 2, but relies on Atlassian Cloud’s certified infrastructure.

Support & Contact

For any questions related to privacy, security, or compliance, please contact support@app-rl.com.

Updates to this Trust Center

This Trust Center may be updated periodically to reflect changes in the app’s functionality or compliance posture. Continued use of the app implies acceptance of the latest version of these policies.
